Tension Policies

How the Braid handles strands running at different speeds.

TightAtomic lock-step

Advances only when all strands yield. Slower strand holds the faster one until it catches up.

fiber-A
fiber-B
Output — synchronized tuples
Tuples: 0State: idle
const braid = new Braid([fiberA, fiberB]); // Tension.Tight is the default for await (const [a, b] of braid) { ... }
LooseFluid / cached

Fast strands use the last cached value of slow ones. Emits whenever any strand yields.

fast 120ms
slow 900ms
Output — fluid, slow value is cached
Fast: 0 Slow: 0 Emits: 0
const braid = new Braid([fast, slow], { tension: Tension.Loose });
FrayedStrict / fails fast

If any strand exhausts or throws, the entire Braid stops immediately. Strict pipelines only.

long (∞)
short (5)
Output — halts when short strand ends
Status: idle
const braid = new Braid([long, short], { tension: Tension.Frayed }); // stops when 'short' ends
Bind & ComposeFiber of Braids

braid.bind(fn) returns a new Fiber — which enters another Braid. Infinite nesting, no special API.

Level 1: [ask, bid] → spread
Level 2: [spread, signal] → decision
const spread = new Braid([ask, bid]) .bind(([a,b]) => a - b); // Fiber<number> const decision = new Braid([spread, sig]) .bind(([sp, s]) => sp > 2 ? s : "WAIT");

Combinators

Free functions that reshape and combine Fibers.

merge()Fan-in

Emit from N Fibers as values arrive — unordered, non-blocking. Three sources race below.

Merged output (arrival order, color = source)
const stream = merge(fiberA, fiberB, fiberC); // values arrive race-style, unordered
map · filter · takeFiber ops

Chain transformations fluently on a Fiber. Each op returns a new Fiber. Watch a counter get filtered and mapped.

counter(0) → filter(even) → map(×10) → take(8)
counter(0) .filter(n => n % 2 === 0) .map(n => n * 10) .take(8); // [0,20,40,60,80,100,120,140]
